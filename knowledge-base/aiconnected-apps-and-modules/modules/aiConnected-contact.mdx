```
# Contact by aiConnected

Contact by aiConnected is an intelligent AI-powered contact form system designed to transform website inquiries into action. Unlike traditional forms that passively collect messages, Contact actively understands, routes, responds to, and escalates inquiries in real-time—helping businesses capture more leads, close deals faster, and reduce missed opportunities.

---

## ✨ Key Features

- 🔎 **AI Message Understanding**  
  Classifies messages as Sales, Support, Billing, Press, etc., with intent and urgency detection.

- ⚡ **Real-Time Routing**  
  Routes messages instantly to the correct team, CRM, helpdesk, or calendar based on AI classification.

- 💬 **Smart Auto-Replies**  
  Responds to inquiries with context-aware messages using OpenAI, Claude, or local LLMs.

- 📞 **Voice Message Support**  
  Visitors can leave voice notes that are transcribed and processed using ElevenLabs, Deepgram, or Whisper.

- 🧠 **Lead Scoring + Urgency Alerts**  
  Identifies high-priority leads and instantly notifies stakeholders via Slack, SMS, or Email.

- 🚫 **AI Spam Filtering**  
  Blocks bots and spam using behavior-based fingerprinting and natural language models.

- 📊 **Analytics & Logging**  
  Tracks inquiry types, response outcomes, spam ratios, and conversion metrics.

```

## **🔁 Step-by-Step Breakdown of How It Works**

---

### **1\. Form Interaction (Frontend Layer)**

The user interacts with a modern, customizable form widget embedded on your site. This form supports:

* Name, email, phone, message, and any custom fields (e.g., service type, product category, etc.)

* Optional voice message (recorded via browser mic API)

* Real-time smart typing detection (non-invasive, for early AI pre-processing)

* Optional reCAPTCHA alternatives for spam protection (honeypots \+ fingerprinting)

**Frontend Technologies:**

* React or WebComponent wrapper

* Uses a POST hook to the backend (or direct to an n8n webhook)

* JSON payload includes metadata: device, timestamp, source URL, UTM if present

---

### **2\. Message Reception & AI Classification (Mid-layer Logic)**

Once the form is submitted, Contact initiates a webhook to its internal **n8n workflow engine**, triggering the following logic stack:

#### **a. Preprocessing Node**

* Clean the message text (remove junk characters, normalize case, etc.)

* If voice was included, convert audio → text via ElevenLabs, Whisper, or Deepgram

#### **b. AI Message Understanding Node**

* Uses a local or cloud LLM (GPT-4, Claude, or Qwen 32B) to:

  * Summarize the inquiry

  * Classify it (Sales / Support / Billing / Press / Other)

  * Extract structured entities (Name, Company, Budget, Dates, etc.)

  * Identify sentiment (angry, happy, confused, etc.)

  * Score urgency (e.g., “I need help now” vs “Just curious”)

  * Detect spam likelihood (based on phrasing, links, patterns, user history)

#### **c. AI Lead Scoring Node**

* If message fits a “sales” or “consulting” classification:

  * Score the lead on a 0–100 scale based on criteria like:

    * Budget mentioned

    * Buying intent signals

    * Industry match

    * Size of opportunity

  * Mark “high intent” leads for priority routing

---

### **3\. Routing Logic & Business Action Nodes**

Now that the message is understood and classified, Contact performs conditional actions:

#### **a. Spam Handling**

* Messages with a spam score \> 85 are silently flagged or deleted

* Messages with moderate score (50–85) are routed to a “review” inbox

* Optional CAPTCHA fallback can be triggered on next form visit

#### **b. Message Routing**

Based on classification and extracted keywords:

* **Sales inquiries** → routed to CRM (e.g., GoHighLevel, HubSpot, Pipedrive)

* **Support requests** → converted to a Helpdesk ticket (e.g., Chatwoot, Freshdesk)

* **Billing** → sent to finance queue

* **Recruiting** → routed to HR/PeopleOps inbox

* **Press** → sent to founder or media contact

You can configure this routing logic visually in n8n using "if", "switch", or router nodes. You may also tag team members based on keywords (e.g., “enterprise pricing” → notify BD team).

#### **c. CRM & Calendar Sync**

* If contact info includes email/phone and the message is qualified:

  * Create/update contact record in CRM

  * If AI detects scheduling intent, offer or auto-schedule appointment via Google Calendar, Calendly, or Motion App

  * Follow-up task created for human rep (with full AI summary attached)

---

### **4\. AI-Powered Auto-Reply (Optional)**

If enabled, Contact can generate a response immediately, such as:

* Personalized acknowledgement

* Basic product/service answer

* Scheduling link

* Escalation guidance

This reply is sent by:

* Email (via SendGrid, Gmail API, or SMTP)

* SMS (via Twilio)

* Chat (if user is still active)

All auto-replies are tracked and logged in the conversation history for transparency.

---

### **5\. Live Escalation (Chat/Voice)**

If Contact determines the inquiry is complex, time-sensitive, or unclear, it can:

* Open a live chat box

* Offer a voice conversation with AI (or a human)

* Transfer to a human via Slack, WhatsApp, or phone call

This is handled using integrations like:

* LiveKit or Whisper for real-time AI voice

* Crisp, Tidio, or custom LiveChat SDKs for human handoff

---

### **6\. Logging, Memory, & Feedback Loop**

All interactions are logged into a structured database or Google Sheet:

| Field | Description |
| ----- | ----- |
| Name | Parsed name or email address |
| Message Summary | AI-generated |
| Type | Sales, Support, Billing, etc. |
| Spam Score | 0–100 |
| Urgency Score | 0–100 |
| Lead Score | 0–100 |
| AI Reply | Optional generated response |
| Routing Path | Where the inquiry was sent |
| Timestamp | Submission time |
| Outcome (optional) | Later appended if the message led to a conversion |

This data is used for:

* Ongoing fine-tuning of AI responses

* Spam rule improvement

* Sales attribution and reporting
